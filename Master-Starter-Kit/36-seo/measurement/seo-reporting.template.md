## Monthly Report Template

Delivered within 5 business days of month-end. The monthly report answers: **"What happened in organic search this month, and what are we doing about it?"**

### Report Header

```
SEO MONTHLY REPORT — {{PROJECT_NAME}}
Report Period: {{MONTH}} {{YEAR}}
Prepared by: [SEO Lead Name]
Date: [Report Date]
```

### Section 1: Executive Summary

**Length:** 3-5 bullet points. This is the only section many stakeholders will read.

```
EXECUTIVE SUMMARY — {{MONTH}} {{YEAR}}
──────────────────────────────────────

• Organic traffic: [number] sessions ([+/- X%] vs previous month,
  [+/- X%] vs same month last year)

• Organic conversions: [number] ([+/- X%] vs previous month)
  Revenue attributed to organic: $[number]

• Key wins: [1-2 sentence summary of biggest positive development]

• Key challenges: [1-2 sentence summary of biggest issue or risk]

• Priority for next month: [1 sentence — the single most important
  SEO action for the coming month]
```

**Rules for the executive summary:**
- Lead with numbers, not narratives
- Always include year-over-year comparison (month-over-month is noisy)
- State challenges honestly — hiding problems destroys trust
- The "priority for next month" should be specific and actionable, not vague

## Stakeholder-Specific Variants

Different audiences need different reports. Do not send the 15-page monthly report to the CEO.

### C-Suite Report

**Audience:** CEO, CFO, CMO
**Frequency:** Monthly (1 page) + Quarterly (2-3 pages)
**Format:** Dashboard screenshot or 1-page summary

**What to include:**
- Organic revenue and conversions (the money numbers)
- YoY growth trend (one chart)
- ROI calculation (investment vs return)
- Top-line competitive position (winning or losing market share)
- One strategic recommendation (the single most important ask)

**What to exclude:**
- Individual keyword rankings
- Technical health details
- Backlink counts
- Content freshness metrics
- SERP feature analysis

**Framing:** Business impact language. "Organic search generated $X in revenue this month" not "We improved our keyword visibility by 3%."

### Marketing Team Report

**Audience:** Content team, demand gen, marketing managers
**Frequency:** Monthly (full report) + Weekly (keyword update)
**Format:** Full monthly template + weekly keyword digest

**What to include:**
- All sections of the monthly report
- Detailed keyword performance by topic cluster
- Content performance analysis (which content is driving traffic and conversions)
- Content recommendations (what to write next, what to refresh)
- Competitor content analysis (what competitors published)
- SERP feature opportunities

**What to exclude:**
- Server log analysis details
- Crawl budget metrics
- Core Web Vitals technical details (they need the pass/fail, not the milliseconds)

**Framing:** Content strategy language. "The API rate limiting guide is now #3 for its target keyword and driving 2,400 monthly sessions" not "DR increased by 2 points."

### Engineering Team Report

**Audience:** Engineering lead, web platform team, DevOps
**Frequency:** Monthly (technical section only) + as-needed for issues
**Format:** Technical section of monthly report + specific issue tickets

**What to include:**
- Core Web Vitals detailed breakdown (LCP, INP, CLS by page type, with specific URLs)
- Crawl error analysis (status codes, failing URLs, response times)
- Index coverage issues (with specific URLs and exclusion reasons)
- Rendering issues (JS rendering, dynamic rendering status)
- Server log analysis summary (crawl patterns, response time data)
- Specific technical recommendations with implementation details
- Cross-reference to `36-seo/measurement/crawl-budget-log-analysis.md` for log data

**What to exclude:**
- Keyword rankings
- Content performance
- Backlink profiles
- Competitive analysis
- Revenue attribution

**Framing:** Technical language. Specific URLs, status codes, response times, and implementation steps. Engineers need "fix the 500 error on /api/products returning a database timeout" not "our technical health needs improvement."

## Visualization Recommendations

### What to Chart (Line or Area Chart)

Use line/area charts for metrics where **trends over time** are the insight:
- Organic sessions (daily or weekly, 90-day view)
- Keyword visibility score (weekly, 6-month view)
- Domain rating (monthly, 12-month view)
- Organic CTR (monthly, 12-month view)
- Referring domain count (monthly, running total)

**Best practices:**
- Always include a comparison line (previous period or YoY)
- Use a 7-day moving average for daily data to smooth out weekly cycles
- Annotate major events (algorithm updates, site launches, content publishes) on the timeline
- Keep the y-axis starting at 0 unless the data range makes this impractical

### What to Table

Use tables for metrics where **detail and specificity** are the insight:
- Top landing pages (with sessions, conversions, positions)
- Keyword movers (with old and new positions)
- Backlink acquisitions (with source details)
- Technical issues (with URLs and status codes)
- Action items (with owners and deadlines)

**Best practices:**
- Sort by the most important column (usually traffic impact or priority)
- Limit to 10-20 rows (link to full data if more is needed)
- Use conditional formatting for status columns (green/yellow/red)
- Include the "so what" column — not just the data, but the action or implication

### What to Bullet Point

Use bullet points for **qualitative insights and recommendations:**
- Executive summary
- Competitive intelligence insights
- Strategy recommendations
- Lessons learned
- Risk assessment

**Best practices:**
- Lead each bullet with the conclusion, then the supporting evidence
- Keep each bullet to 1-3 sentences
- Use sub-bullets for supporting detail, not separate top-level bullets
- Number bullets when order/priority matters; use unordered when items are equal weight

### What to Display as a Number Card

Use large number cards (KPI tiles) for **the 4-6 most important metrics:**
- Organic sessions (with % change)
- Organic revenue (with % change)
- Keyword visibility (with change)
- Domain rating (with change)

**Best practices:**
- Include the comparison (% change or absolute change)
- Use directional arrows and color (green up, red down)
- Place at the top of the dashboard — these are the first things seen
- Maximum 6 cards; more than that dilutes attention

### What to Display as a Gauge

Use gauge charts for **metrics with a clear target or threshold:**
- Core Web Vitals pass rate (target: 90%+)
- Index coverage (target: 90%+)
- Content freshness score (target: 80%+)
- Crawl error rate (target: <1%)

### Chart Type Decision Matrix

| Data Type | Recommended Visualization | Example |
|---|---|---|
| Trend over time | Line chart | Organic sessions over 90 days |
| Composition / distribution | Pie or stacked bar | Keyword position distribution |
| Comparison across categories | Horizontal bar | Traffic by landing page |
| Relationship between two variables | Scatter plot | CTR vs position |
| Progress toward target | Gauge | CWV pass rate |
| Single important number | KPI card | Monthly organic revenue |
| Detailed multi-dimensional data | Table | Keyword performance details |
| Before/after comparison | Grouped bar | This month vs last month by segment |

## BI Pipeline Integration

For automated reporting, connect SEO data sources to the BI infrastructure documented in `35-business-intelligence/`.

### Automated Report Generation

Instead of manually pulling data each month, automate the data pipeline:

1. **GA4 > BigQuery** — automatic export (free). Raw organic traffic data available for SQL analysis.
2. **Search Console > BigQuery** — bulk data export. Query and page performance data.
3. **Ahrefs/SEMrush API > data warehouse** — scheduled daily/weekly pulls for ranking and backlink data.
4. **{{DASHBOARD_TOOL}} connected to data warehouse** — live dashboards that update automatically.

**What to automate:**
- Data collection and aggregation (eliminate manual data pulls)
- Dashboard updates (real-time or daily refresh)
- Alert triggers (threshold-based notifications)

**What NOT to automate:**
- The executive summary (requires human judgment and narrative)
- Strategic recommendations (requires context that data alone cannot provide)
- Competitive intelligence insights (requires interpretation)
- The "so what" for each data point (the most valuable part of the report)
