# Design Review System

## Purpose

A standalone HTML-based design review portal that gets scaffolded into every project generated by the Master Starter Kit. It provides a structured workflow for reviewing, rating, and approving design variations before implementation begins.

## Workflow

```
Generate Designs → Generate Thumbnails → Review in Gallery → Full-Screen Review → Export JSON → AI Analysis → Design Direction Doc
(/design-generate)   (/review:thumbs)     (index.html)        (review.html)       (/review:export)   (/review:report)
```

## What's Included

| File | Purpose |
|------|---------|
| `templates/index.html` | Thumbnail gallery with filtering, rating, approve/reject, summary stats, JSON export |
| `templates/review.html` | Full-screen design viewer with collapsible side panel for detailed review |
| `templates/generate-thumbnails.mjs` | Node.js/Playwright script to auto-screenshot HTML design files |
| `templates/gen-thumbs.sh` | Shell script alternative using Playwright CLI |
| `review-config.template.md` | Configuration reference for `reviewSystem` settings in project config |
| `ai-analysis-prompt.template.md` | Prompt template for AI analysis of exported review JSON |

## Kit Commands

| Command | What It Does |
|---------|-------------|
| `/review:init` | Scaffolds the review system into the current project |
| `/review:thumbs` | Runs Playwright to generate/refresh all design thumbnails |
| `/review:export` | Reads localStorage export JSON and formats as markdown for AI |
| `/review:report` | Generates a design direction report from review data |

## Configuration

Added to the project's kit config (CONFIG in STATE BLOCK):

```
REVIEW_SYSTEM: {
  enabled: true,
  designsPath: "designs/lookfeels/",
  outputPath: "design-review/",
  screens: [
    { key: "dashboard", label: "Dashboard", file: "dashboard.html", color: "#6366f1" },
    { key: "list", label: "List View", file: "list.html", color: "#10b981" }
  ],
  styles: "auto",           // "auto" = detect from folder structure, or explicit array
  elementsPerScreen: {
    dashboard: ["header", "kpi-cards", "chart", "sidebar", "activity-feed"],
    list: ["table", "filters", "search", "pagination", "bulk-actions"]
  }
}
```

## Integration with Kit Phases

The design review system creates a new **Phase 0: Design Review** that runs after design generation but before Phase 1 (Token Foundation):

```
Design Pipeline                     Kit Development Phases
═══════════════                     ═════════════════════
/design-research
/design-generate  ──→  Phase 0: Design Review  ──→  Phase 1: Token Foundation
                       (this system)                 (design tokens from approved styles)
```

The exported review data feeds directly into token decisions — approved color palettes, spacing preferences, and component styles become the basis for design tokens.

## How It Works

### Gallery (index.html)
- Grid of thumbnail cards showing each style × screen combination
- Filter by screen type, style, status (pending/approved/rejected)
- Search by name
- Progress tracker showing how many designs have been reviewed
- Click any card to open the full-screen reviewer
- Export all review data as JSON

### Full-Screen Reviewer (review.html)
- Left: iframe showing the actual HTML design at full size
- Right: collapsible 340px panel with review controls
- Star rating (1-5), approve/reject buttons
- Per-screen element tagging (liked/disliked individual elements)
- Notes textarea for detailed feedback
- Keyboard shortcuts for fast reviewing

### Keyboard Shortcuts
| Key | Action |
|-----|--------|
| ← → | Navigate between designs |
| 1-5 | Set star rating |
| A | Approve current design |
| R | Reject current design |
| P or Esc | Toggle side panel |
| N | Focus notes textarea |

### Data Persistence
- All review data stored in localStorage
- Shared between gallery and reviewer via same storage key
- JSON export includes: ratings, approvals, element preferences, notes, timestamps
- Export format designed for AI consumption (structured, with stats)

## Tech Stack
- Standalone HTML (no build step)
- Tailwind CSS via CDN
- Inter font via Google Fonts
- Playwright for thumbnail generation (Node.js)
- localStorage for persistence
- Works with `file://` protocol
