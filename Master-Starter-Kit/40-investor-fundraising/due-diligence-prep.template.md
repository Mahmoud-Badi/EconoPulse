## 2. Corporate Documents Checklist

- [ ] **Certificate of Incorporation** — current, including all amendments (state-filed)
- [ ] **Bylaws** — current version, signed by board
- [ ] **Board Resolutions** — all resolutions since incorporation
- [ ] **Written Consents** — all written consents in lieu of meetings
- [ ] **Board Meeting Minutes** — all meetings since incorporation
- [ ] **Stockholder Agreements** — any voting agreements, right of first refusal, co-sale agreements
- [ ] **Good Standing Certificate** — current, from state of incorporation
- [ ] **Foreign Qualifications** — registrations in states where you operate (if applicable)
- [ ] **EIN Confirmation** — IRS Employer Identification Number letter
- [ ] **Organizational Chart** — legal entity structure (parent, subsidiaries if any)

### Common Gaps to Fix Before Due Diligence

| Gap | Risk Level | Fix |
|---|---|---|
| Missing board resolutions for equity grants | HIGH | Draft and sign retroactive resolutions with legal counsel |
| No IP assignment from contractors | HIGH | Execute Invention Assignment Agreements immediately |
| Operating in states without registration | MEDIUM | File foreign qualifications |
| Missing 83(b) elections | CRITICAL | Cannot be fixed retroactively — tax liability exposure |
| Unsigned employee agreements | HIGH | Execute immediately — every employee, contractor, advisor |

## 4. Product & Technology Documents Checklist

- [ ] **Product Overview** — what the product does, who it serves, key workflows
- [ ] **Demo Video** — 3-5 minute walkthrough of core functionality
- [ ] **Architecture Diagram** — high-level system architecture, infrastructure, and key dependencies
- [ ] **Technology Stack** — languages, frameworks, databases, cloud providers, key SaaS dependencies
- [ ] **Product Roadmap** — 6-12 month roadmap with prioritized features and themes
- [ ] **Technical Debt Assessment** — known technical debt and plan to address
- [ ] **Scalability Plan** — how the system handles 10x and 100x current load
- [ ] **Security Practices** — authentication, encryption, access controls, vulnerability management
- [ ] **Compliance Certifications** — SOC 2, GDPR, HIPAA, PCI-DSS (as applicable)
- [ ] **Uptime / Reliability Data** — SLA commitments, historical uptime, incident response
- [ ] **Open Source Dependencies** — license audit, critical dependency risk assessment
- [ ] **Data Privacy** — data handling practices, data processing agreements, privacy policy

### Technology Risk Matrix

| Risk | Severity | Likelihood | Mitigation |
|---|---|---|---|
| Key-person dependency (single developer owns critical system) | High | | Cross-training, documentation |
| Single cloud provider lock-in | Medium | | Abstraction layers, multi-cloud readiness |
| Scaling bottleneck at [component] | High | | Architecture plan for [specific solution] |
| Open source license risk | Low | | License audit completed |
| Data breach / security incident | High | | Security practices, insurance, incident response plan |

## 5. Legal & IP Documents Checklist

- [ ] **IP Assignment Agreements** — signed by every founder, employee, and contractor who has contributed to the product
- [ ] **Patent Applications** — filed or granted, with status
- [ ] **Trademark Registrations** — filed or granted
- [ ] **Copyright Registrations** — if applicable
- [ ] **Material Contracts** — all contracts representing >5% of revenue or critical to operations
- [ ] **Customer Agreements** — template and any non-standard agreements
- [ ] **Vendor Agreements** — key vendor contracts (hosting, APIs, tools)
- [ ] **Partnership Agreements** — any revenue-sharing, distribution, or strategic agreements
- [ ] **Terms of Service** — current, reviewed by counsel
- [ ] **Privacy Policy** — current, compliant with applicable regulations (GDPR, CCPA)
- [ ] **Employee Offer Letters** — template, with IP assignment clause
- [ ] **Contractor Agreements** — template, with IP assignment and non-compete (where enforceable)
- [ ] **Non-Disclosure Agreements** — executed NDAs relevant to business operations
- [ ] **Pending Litigation** — disclosure of any threatened, pending, or settled litigation
- [ ] **Insurance Policies** — D&O, E&O, general liability, cyber liability
- [ ] **Regulatory Filings** — any required industry-specific filings or licenses

## Pre-Data Room Audit Checklist

Run this before granting any investor access:

- [ ] All documents are current (no drafts, no outdated versions)
- [ ] File names follow consistent naming convention (no "Final_v3_REAL_FINAL.pdf")
- [ ] All financial data is internally consistent (P&L matches revenue detail)
- [ ] Sensitive information is appropriately redacted (customer names if needed)
- [ ] Every folder has at least one document (no empty folders)
- [ ] Document dates are visible (investors want to know how recent data is)
- [ ] Spell-check and formatting review completed on key documents
- [ ] Access permissions are correct (read-only, no edit access for investors)
- [ ] Download tracking is enabled
- [ ] One person is designated as data room owner for updates and questions
