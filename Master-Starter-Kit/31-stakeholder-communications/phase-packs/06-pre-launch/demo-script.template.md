## Fallback Plan

> If something does not work during the demo, use these contingencies.

| Problem | Fallback Action |
|---------|----------------|
| Demo environment is down | Switch to {{FALLBACK_ENVIRONMENT_URL}}. If both are down, use prepared screenshots in {{FALLBACK_SLIDES_LOCATION}}. |
| Login does not work | Use backup account: {{FALLBACK_ACCOUNT}}. If all accounts fail, present from pre-recorded video: {{FALLBACK_VIDEO_LINK}}. |
| Feature X does not load | Skip to next scene. Say: "We have a minor environment issue with this feature. Let me show you the next area and we will circle back." |
| Slow performance | Acknowledge: "The demo environment is running on reduced resources. Production performance targets are {{PROD_PERF_TARGET}}." |
| Data looks wrong | Say: "This is test data — let me walk you through what this would look like with real data." Use {{FALLBACK_DATA_DESCRIPTION}}. |
| Unexpected error | Stay calm. Say: "Interesting — let me note this and move on. This is exactly the kind of thing we catch before launch." Proceed to next scene. |
