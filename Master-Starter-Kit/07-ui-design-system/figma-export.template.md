# Figma / Style Dictionary Token Export — {{PROJECT_NAME}}

## Purpose

Transform design tokens from markdown format into machine-readable JSON that Figma plugins and Style Dictionary can consume. This bridges the gap between the kit's design system output and the tools designers actually use.

## Input

The design tokens file generated by the ORCHESTRATOR (typically `dev_docs/foundations/design-tokens.md` or `dev_docs/foundations/DESIGN-TOKENS.md`).

## How to Use in Figma

### Option A: Tokens Studio Plugin (Recommended)

1. Install "Tokens Studio for Figma" plugin
2. In plugin settings, point to `tokens.json` (via GitHub sync or local file)
3. Tokens appear as Figma styles automatically
4. Changes to `tokens.json` sync back to Figma

### Option B: Style Dictionary CLI

```bash
npx style-dictionary build --config style-dictionary.config.json
```

This generates platform-specific output:
- CSS custom properties (`tokens.css`)
- Tailwind theme extension (`tailwind-tokens.js`)
- iOS Swift constants
- Android XML resources

### Option C: Manual Import

Copy token values directly from `tokens.json` into Figma's local styles. Less automated but works without plugins.

## Generation Process

1. Read the design tokens markdown file
2. Parse color scales, spacing, typography, and border radius values
3. Map to Style Dictionary format (nested objects with `value` + `type`)
4. Write to `dev_docs/foundations/tokens.json`
5. Validate: every token has a `value` and `type` field
