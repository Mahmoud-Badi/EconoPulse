# Canned Responses — {{PROJECT_NAME}}

> Pre-written responses save time and ensure consistency — but they must sound human. A robotic canned response damages your brand more than a slow personal one.

---

## How to Use These Templates

Every response below has a `[PERSONALIZE]` placeholder. This is where the agent adds specific context from the user's situation. A canned response without personalization is a form letter. A canned response with personalization is a fast, consistent, human reply.

**Rules:**
1. **Always fill in the `[PERSONALIZE]` sections.** Never send a response without customizing it.
2. **Match the user's tone.** If they are casual, be casual. If they are formal, be formal.
3. **Read the user's message fully before selecting a template.** The wrong canned response is worse than a slow manual one.
4. **Modify freely.** These are starting points, not scripts. Change wording to fit the situation.
5. **Sign with your name.** Users should know they are talking to a real person.

## Response Quality Checklist

Before sending any canned response, verify:

- [ ] All `[PERSONALIZE]` placeholders have been filled with specific context
- [ ] The response actually answers the user's question (re-read their message)
- [ ] Tone matches the user's tone (casual/formal)
- [ ] Links are correct and working
- [ ] No placeholder text or template artifacts left in the message
- [ ] Ticket number is included where relevant
- [ ] Agent name is included
- [ ] Response is concise — 2-4 sentences per section, not paragraphs

---

## Creating New Canned Responses

When you find yourself writing the same response more than 3 times:

1. Draft the response with `[PERSONALIZE]` placeholders
2. Have another agent review it for tone and completeness
3. Add it to this document under the appropriate category
4. Add it to your support platform's canned response library
5. Tag it with the relevant ticket category for easy lookup
