# /review:report — Generate Design Direction Report

Reads the design review export JSON and uses AI to generate a comprehensive design direction document. This document feeds directly into Step 13.2 (Token Foundation) as the authoritative source for design token decisions.

## Prerequisites

- Design review export exists (`/review:export` completed, or raw JSON available)
- Review summary at `dev_docs/foundations/design-review-summary.md` (generated by `/review:export`)

## Configuration

- **Export file:** `{{REVIEW_OUTPUT_PATH}}/design-review-export.json`
- **Summary file:** `dev_docs/foundations/design-review-summary.md`
- **Prompt template:** `Master-Starter-Kit/39-review-system/ai-analysis-prompt.template.md`
- **Output file:** `dev_docs/foundations/design-direction.md`

## Steps

### Step 1: Read Inputs

1. Read the export JSON: `{{REVIEW_OUTPUT_PATH}}/design-review-export.json`
2. Read the review summary: `dev_docs/foundations/design-review-summary.md`
3. Read the AI analysis prompt template: the kit's `ai-analysis-prompt.template.md`
4. Read the brand identity worksheet: `dev_docs/foundations/brand-identity.md` (if exists)
5. Read existing design tokens: `dev_docs/foundations/design-tokens.md` (if exists)

### Step 2: Prepare Analysis Context

Combine all inputs into a single analysis prompt:

1. Fill the AI analysis prompt template with:
   - `{{PROJECT_NAME}}` from CONFIG
   - `{{TOTAL_DESIGNS}}` from export metadata
   - `{{SCREEN_COUNT}}` and `{{STYLE_COUNT}}` from export metadata
   - `{{REVIEW_JSON}}` with the full export JSON

2. Append additional context:
   - Brand identity notes (personality, target audience, industry)
   - Any existing design token decisions
   - Anti-slop rules from `07-ui-design-system/anti-slop-rulebook.md`

### Step 3: Generate Design Direction

Using the assembled prompt, generate the design direction document with these sections:

1. **Executive Summary** — Overall recommendation in 2-3 sentences
2. **Style Rankings** — Each style ranked with ADOPT/ADAPT/REJECT verdict
3. **Per-Screen Winners** — Best style for each screen type with rationale
4. **Element Preference Map** — Liked/disliked element patterns across all reviews
5. **Composite Direction** — The MIX of best elements from different styles
6. **Token Implementation Plan** — Concrete design token values extracted from approved designs
7. **Open Questions** — Areas needing human discussion

### Step 5: Write Output

Save to `dev_docs/foundations/design-direction.md`:

```markdown
# Design Direction — {{PROJECT_NAME}}

Generated from design review data on {{DATE}}.
Based on {{REVIEWED}} reviews across {{STYLES}} styles and {{SCREENS}} screens.

{Full analysis from Step 3}

---

## Token Decisions (for Step 13.2)

{Extracted tokens from Step 4}

---

*Generated by /review:report from design review export data.*
*This document is the input for Step 13.2 (Token Foundation).*
```

### Step 6: Report

```

```

## Notes

- The design direction is a RECOMMENDATION, not a mandate. The user reviews and adjusts before token implementation.
- The composite direction (mixing elements from different styles) is typically more valuable than adopting one style wholesale.
- If reviews are incomplete, the report will note confidence levels per screen.
- Re-run this command after additional reviews are completed for updated analysis.
- The token decisions section provides starting values — Step 13.2 refines them into the full token system.
