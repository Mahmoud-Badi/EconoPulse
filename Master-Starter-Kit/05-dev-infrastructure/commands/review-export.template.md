# /review:export — Export Review Data for AI Analysis

Reads the design review JSON export and formats it as a markdown summary suitable for AI consumption. The export is generated by clicking "Export JSON" in the gallery (index.html), which saves a `design-review-export.json` file.

## Prerequisites

- Design reviews completed (or partially completed) in the browser
- Export JSON file exists (downloaded from gallery)

## Configuration

- **Export file:** `{{REVIEW_OUTPUT_PATH}}/design-review-export.json` (or user-specified path)
- **Output file:** `dev_docs/foundations/design-review-summary.md`

## Steps

### Step 1: Locate Export File

```bash
ls {{REVIEW_OUTPUT_PATH}}/design-review-export.json 2>/dev/null || ls ~/Downloads/design-review-export.json 2>/dev/null
```

If not found, instruct the user:
> "Open {{REVIEW_OUTPUT_PATH}}/index.html in your browser, click 'Export JSON' in the top bar, and save the file to {{REVIEW_OUTPUT_PATH}}/design-review-export.json"

### Step 2: Read and Parse Export

Read the JSON file. Extract:

- **Metadata**: project name, export date, total designs
- **Summary stats**: average rating, approval rate, total reviewed
- **Per-style breakdown**: average rating, approval count, rejection count per style
- **Per-screen breakdown**: winning style per screen, rating distribution
- **Element trends**: most liked elements, most disliked elements
- **All notes**: concatenated reviewer notes

### Step 3: Generate Markdown Summary

Write to `dev_docs/foundations/design-review-summary.md`:

```markdown
# Design Review Summary — {{PROJECT_NAME}}

Exported: {{EXPORT_DATE}}
Reviewed: {{REVIEWED_COUNT}} / {{TOTAL_DESIGNS}} designs ({{REVIEW_PERCENT}}%)

## Overall Stats

| Metric | Value |
|--------|-------|
| Average Rating | {{AVG_RATING}} / 5 |
| Approved | {{APPROVED_COUNT}} ({{APPROVED_PERCENT}}%) |
| Rejected | {{REJECTED_COUNT}} ({{REJECTED_PERCENT}}%) |
| Pending | {{PENDING_COUNT}} |

## Style Rankings

| Rank | Style | Avg Rating | Approved | Rejected | Verdict |
|------|-------|-----------|----------|----------|---------|
| 1 | {{STYLE}} | {{RATING}} | {{N}} | {{N}} | {{ADOPT/ADAPT/REJECT}} |
| ... | | | | | |

## Per-Screen Winners

| Screen | Winning Style | Rating | Runner-Up | Notes |
|--------|--------------|--------|-----------|-------|
| {{SCREEN}} | {{STYLE}} | {{RATING}} | {{STYLE}} | {{NOTES}} |

## Element Preferences

### Most Liked
{{LIST_OF_LIKED_ELEMENTS}}

### Most Disliked
{{LIST_OF_DISLIKED_ELEMENTS}}

### Style-Specific Wins
{{ELEMENTS_THAT_WORKED_IN_SPECIFIC_STYLES}}

## Reviewer Notes (Raw)

{{ALL_NOTES_CONCATENATED}}

---

*This summary was generated from design review data. Feed to AI with the analysis prompt from `39-review-system/ai-analysis-prompt.template.md` for a full design direction document.*
```

### Step 4: Confirm Output

```
REVIEW DATA EXPORTED
════════════════════
Source: {{EXPORT_FILE}}
Output: dev_docs/foundations/design-review-summary.md

Stats:
  Designs reviewed: {{N}} / {{TOTAL}}
  Average rating:   {{AVG}} / 5
  Top style:        {{STYLE}} ({{RATING}} avg)
  Approval rate:    {{PERCENT}}%

Next: Run /review:report to generate the full AI design direction analysis.
```

## Notes

- The export JSON is the raw data from localStorage. This command transforms it into a readable markdown summary.
- The summary is an intermediate artifact — the full design direction document is generated by `/review:report`.
- If reviews are incomplete (< 100%), the summary will note this and flag which designs are unreviewed.
- Multiple reviewers can export separately. To merge, combine the JSON files and take the latest review per design.
